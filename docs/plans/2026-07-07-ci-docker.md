## Tasks

### - [x] Task 1: 前端支持同源部署

**Files:** `frontend/src/lib/api-client.ts`
**Steps:** 第 3 行 `||` 改为 `??`,使 `VITE_API_URL=""` 表示同源相对路径;未设置时行为不变。
**Verify:** `bun run lint && bun run test:run`

### - [x] Task 2: 后端服务前端静态产物

**Files:** `backend/app/main.py`
**Steps:** 所有 `include_router` 之后,若 `backend/static/` 存在:挂载 `/assets` 静态资源 + GET catch-all SPA fallback(存在的文件直接返回,否则 `index.html`)。
**Verify:** `uv run pytest && uv run ruff check . && uv run mypy .`(本地无 static 目录,行为零变化)

### - [x] Task 3: Dockerfile + .dockerignore + docker-compose.yml

**Files:** `Dockerfile`、`.dockerignore`、`docker-compose.yml`(仓库根)
**Steps:** 多阶段构建(bun build 前端 → python:3.12-slim + uv 装后端依赖 → 拷入 dist 到 `backend/static/`);compose 单 service,卷挂 `./data:/app/backend/data`。
**Verify:** `docker compose up --build`,curl `/health`,浏览器走通「搜索基金→加自选→看详情」,重启容器数据仍在。

### - [x] Task 4: GitHub Actions workflow

**Files:** `.github/workflows/ci.yml`
**Steps:** backend(uv + ruff/mypy/pytest)、frontend(bun + lint/vitest/build)、docker(buildx + gha 缓存,main push 推 GHCR)三个 job;concurrency cancel-in-progress;timeout-minutes。
**Verify:** 推 PR 三 job 全绿;合 main 后 GHCR 出现镜像。

### - [x] Task 5: 文档收尾

**Files:** `README.md`、`docs/ROADMAP.md`
**Steps:** README 加 CI badge 与 Docker 部署一节;ROADMAP 第 6 项更新。
**Verify:** 人工核对。
